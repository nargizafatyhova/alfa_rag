# Alfa-Bank x МФТИ — RAG-ответы по базе знаний

Пайплайн, который по вопросу пользователя ищет релевантные фрагменты в корпусе страниц
`alfabank.ru` и генерирует ответ на их основе. Всё на open-source моделях, инференс локальный,
без внешних API генерации.

## Метрика

Оценка — `Recall-L = BERTScore_recall(ответ, эталон) * L(q)`, где `L(q)` мягко штрафует длину:
единица при `l_a <= 1.5 * l_эталона`, линейно падает до нуля на `3 * l_эталона`.

Что из этого следует и определило решение:

- Критична калибровка длины: ответ держим около длины эталона (целимся в `1.2–1.4x`,
  не больше `1.5x`). Переход от длинных ответов к длине эталона дал основной прирост скора.
- Решение «есть ли ответ» лучше отдавать самой LLM, читающей контекст: bi-encoder косинус почти
  не отличает отвечаемые вопросы от неотвечаемых, cross-encoder отличает заметно лучше.
- `metric.py` — своя реализация метрики на русском BERT

## Пайплайн

| Этап | Что делаем | Файл |
|---|---|---|
| Чанкинг | режем страницы на куски ~320 токенов с overlap, добавляем title | `preprocess.py` |
| Поиск | dense `BAAI/bge-m3` + BM25 по леммам, слияние через RRF | `retrieval.py`, `step1_retrieve.py` |
| Реранк | `BAAI/bge-reranker-v2-m3` | `step2_rerank.py` |
| Генерация | `Qwen2.5-14B-Instruct` (fp16, vLLM) + калибровка длины | `step3_generate.py` |
| Метрика | BERTScore-recall x L(q) для локальной оценки | `metric.py` |

Альтернативный (более тяжёлый) вариант: HyDE-поиск по гипотетическому ответу
(`hyde_retrieve.py`) и генерация на `Qwen2.5-32B-Instruct-GPTQ-Int4` (`gen_32b.py`).

## Запуск

```bash
pip install -r requirements.txt

# рядом нужны websites.csv и questions.csv
python step1_retrieve.py   # чанкинг + поиск, пишет candidates.parquet
python step2_rerank.py     # реранк, пишет reranked.parquet
python step3_generate.py   # генерация + калибровка длины, пишет submission.csv
```

Нужна GPU (для bge-m3, реранкера и vLLM). 14B в fp16 требует ~28 ГБ или TP=2 на двух картах.
Ретрив для отладки можно гонять на CPU с малыми моделями (например `multilingual-e5-small` в
`retrieval.py`).

Тяжёлый вариант поверх первого прохода:

```bash
python hyde_retrieve.py    # ищет по вопрос+ответ первого прохода, пишет reranked2.parquet
python gen_32b.py          # 32B генерация, пишет answers_32b.csv
```
